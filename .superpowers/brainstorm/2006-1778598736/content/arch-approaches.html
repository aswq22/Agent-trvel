三种多 Agent 架构方案

同样是分阶段混合模式，组织方式不同，选你最认可的

A

平铺节点图（Flat Graph）

[用户] → [解析Agent] → [景点Agent]
                        ↓
       [路线Agent] [酒店Agent] [美食Agent] ←并行
                    ↓
             [攻略Agent（汇总）]

一个 LangGraph，所有 Agent 都是节点，用 Send() API 实现并行分发。改造成本最低，沿用现有代码结构。

优点

  • 改动量最小
  • 调试直观
  • LangGraph 原生支持

缺点

  • 所有逻辑堆在一个图里
  • 扩展新 Agent 需改主图
B

Supervisor 模式

[用户] → [Supervisor Agent]
              ↙ ↓ ↓ ↓ ↘
   [景点] [路线] [酒店] [美食] [攻略]
              ↖ ↑ ↑ ↑ ↗
          [Supervisor 整合输出]

Supervisor LLM 决定调用哪个 Worker，动态路由。源自 LangGraph 官方 Supervisor 教程模式。

优点

  • 动态决策，更智能
  • 易插拔新 Worker
  • 符合主流 Multi-Agent 范式

缺点

  • Supervisor 调用次数多，token 消耗大
  • 调试路由逻辑较复杂
C

子图嵌套（Subgraph）⭐ 推荐

主图: [解析] → [景点子图] → [并行子图组] → [攻略子图]
                        ↕
       [路线子图] [酒店子图] [美食子图]

每个 Agent 是独立的 LangGraph 子图，各有自己的 State、工具和提示词。主图通过 Send() 并行调度子图。

优点

  • 模块化最强，各 Agent 完全独立
  • 每个子图可单独测试
  • 最能展示多智能体架构深度
  • 扩展新 Agent 无需改主图

缺点

  • 实现工作量最大
  • State 跨子图传递需设计

💡 推荐 C：子图嵌套 — 每个 Agent 有独立生命周期，架构最清晰，最能体现多智能体系统的设计深度
