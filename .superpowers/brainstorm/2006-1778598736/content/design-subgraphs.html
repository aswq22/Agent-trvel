各子图内部设计

每个 Agent 子图都遵循同一结构：工具调用 → LLM 推理 → 结构化输出

统一子图模式（所有 Agent 共用）
[子图入口 State 切片] → tool_node（MCP 工具调用）→ llm_node（Qwen 推理）→ output_node（结构化写回）
各 Agent 专属配置
🧩 ParserAgent（主图节点，非子图）
输入：user_input (str)
工具：无（纯 LLM 解析）
输出：TripParams 结构体
提示词引导 LLM 从自然语言提取
destination / days / budget / preferences
🏛 AttractionAgent 子图
输入：trip_params
工具：gaode_poi_search, dianping_attraction_search
输出：attractions: List[Attraction]
按偏好筛选，输出景点名/评分/地址/推荐理由
🗺 RouteAgent 子图
输入：trip_params + attractions
工具：gaode_route_plan, gaode_distance_matrix
输出：route: RouteInfo（按天分组）
计算景点间距离，生成每日行程顺序和交通方式
🏨 HotelAgent 子图
输入：trip_params + attractions（确定住宿区域）
工具：ctrip_hotel_search, ctrip_hotel_detail
输出：hotels: List[Hotel]（含价格/评分/位置）
按预算和景点位置推荐，每晚提供 2-3 个选项
🍜 FoodAgent 子图
输入：trip_params + attractions（按区域匹配）
工具：dianping_restaurant_search, dianping_menu
输出：foods: List[Restaurant]（含特色菜/价位/位置）
按口味偏好和景点附近推荐早/午/晚餐
📋 StrategyAgent 子图
输入：全部 State（attractions + route + hotels + foods）
工具：无（纯综合推理）
输出：final_plan (str)，SSE 流式推送
按天生成完整攻略：早/午/晚行程 + 住宿 + 注意事项
错误处理策略
MCP 工具失败 → 子图内 retry 1次，仍失败则 LLM 用内置知识降级回答，不中断主流程
并行子图失败 → 记录 error 字段到 State，StrategyAgent 在攻略中标注"该部分暂不可用"
解析失败 → ParserAgent 返回默认 TripParams，提示用户补充信息
